# Security Policy

## Supported Versions

This project is maintained from the default branch. Security fixes should target the current code unless a release branch is explicitly created.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately by opening a GitHub security advisory for this repository, or by contacting the maintainer through the repository owner's GitHub profile.

Do not open a public issue that includes exploit details, private media, credentials, tokens, or links to sensitive files.

## What To Include

- A concise description of the issue.
- Steps to reproduce with synthetic or non-sensitive media.
- The affected command, platform, dependency, or model path.
- Any impact you believe the issue may have.

## Scope

Security reports may include dependency vulnerabilities, unsafe file handling, command execution concerns, privacy leaks, or behavior that could expose source or target media unexpectedly.

Misuse reports involving impersonation, harassment, non-consensual intimate imagery, or deceptive media should also follow the [Responsible Use](RESPONSIBLE_USE.md) guidance.
